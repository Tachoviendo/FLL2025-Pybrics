# COMENTARIOS POR TODOS LADOS


## ¿Qué son los comentarios?
Si usamos `#` en nuestro código, estamos creando comentarios. Son lineas en el código que python va a ignorar cuando lo ejecutemos. Son escenciales para no perdernos y compartir con otros nuestro código

```python
#Esto es un comentario
ruedaIzquierda = Motor(Port.A, Direction.COUNTERCLOCKWISE)
ruedaDerecha = Motor(Port.E, Direction.CLOCKWISE)

#Esto también 
motorDerecho = Motor(Port.F, Direction.CLOCKWISE)
motorIzquierdo = Motor(Port.B, Direction.COUNTERCLOCKWISE) #y esto también es un comentario
```
## PORFAVOR USEN COMENTARIOS
Generalmente, los códigos que  haremos en pybrics, van a ser secuencias de pasos, una atras de otra y si no ponemos con nuestrar palabras que hace cada cosa, terminamos con algo como esto:

```python
robot.straight(200)
robot.turn(60)
robot.straight(200)
robot.turn(60)
robot.straight(200)
robot.turn(60)
robot.straight(200)
robot.turn(60)
robot.straight(200)
robot.turn(60)
robot.straight(200)
robot.turn(60)
```

Si lo leemos con mucho detenimiento, nos damos cuenta que ese código hace que nuestro robot dibuje un hexágono, pero miren ahora:
```python
# Hacer un hexágono, lado por lado
robot.straight(200)
robot.turn(60)

robot.straight(200)
robot.turn(60)

robot.straight(200)
robot.turn(60)

robot.straight(200)
robot.turn(60)

robot.straight(200)
robot.turn(60)

robot.straight(200)
robot.turn(60)
```

Espaciando un poco el código y simplemente poniendo un comentario, cualquiera que lo lea, sabrá que hace nuestro código.
